# 📖 메모 앱 사용자 가이드

## 🎯 사용자별 메모 관리

이 메모 앱은 **각 사용자별로 완전히 독립된 메모 공간**을 제공합니다.

### ✅ 지원되는 기능

#### 1. 회원가입 및 로그인
- 여러 계정을 만들 수 있습니다
- 각 계정은 독립적인 메모 공간을 가집니다
- 이메일과 비밀번호로 안전하게 관리됩니다

#### 2. 사용자별 메모 CRUD
각 사용자는 다음 기능을 사용할 수 있습니다:

##### ✏️ Create (생성)
- **새 폴더 생성**: 메인 화면에서 "새 폴더" 버튼
- **새 메모 생성**: 폴더 선택 후 "새 메모" 버튼
- 생성된 데이터는 현재 로그인한 사용자에게만 표시됩니다

##### 📖 Read (조회)
- **폴더 목록**: 현재 사용자가 만든 폴더만 표시
- **메모 목록**: 선택한 폴더 내의 메모만 표시
- **검색**: 현재 사용자의 메모만 검색
- **최근 메모**: 현재 사용자의 최근 메모 5개 표시

##### ✨ Update (수정)
- **메모 편집**: 실시간 자동 저장 (300ms)
- **폴더 이름 변경**: 추후 지원 예정
- 수정 사항은 현재 사용자의 데이터에만 반영

##### 🗑️ Delete (삭제)
- **메모 삭제**: 휴지통으로 이동
- **폴더 삭제**: 폴더와 포함된 메모 모두 휴지통으로
- **휴지통에서 복원**: 삭제된 메모 복원
- **영구 삭제**: 휴지통에서 완전히 삭제

## 📝 사용 시나리오

### 시나리오 1: 첫 번째 사용자 (홍길동)

```
1. 앱 실행 → 로그인 페이지
2. "회원가입" 탭 클릭
3. 정보 입력:
   - 이름: 홍길동
   - 이메일: hong@example.com
   - 비밀번호: password123
4. "회원가입하기" 클릭 → 자동 로그인 ✅

5. 메모 작성:
   ✏️ "업무" 폴더 생성
   ✏️ "회의록" 메모 작성
   ✏️ "TODO 리스트" 메모 작성

6. 결과:
   📁 업무 폴더
   ├─ 📝 회의록
   └─ 📝 TODO 리스트
```

### 시나리오 2: 두 번째 사용자 (김철수)

```
1. 로그아웃 클릭 🚪
2. "회원가입" 탭에서 새 계정 생성:
   - 이름: 김철수
   - 이메일: kim@example.com
   - 비밀번호: mypass456
3. "회원가입하기" 클릭 → 자동 로그인 ✅

4. 메모 작성:
   ✏️ "개인" 폴더 생성
   ✏️ "일기" 메모 작성
   ✏️ "아이디어" 메모 작성

5. 결과:
   📁 개인 폴더
   ├─ 📝 일기
   └─ 📝 아이디어

   ⚠️ 홍길동의 메모는 보이지 않음!
```

### 시나리오 3: 다시 첫 번째 사용자로 전환

```
1. 로그아웃 클릭 🚪
2. "로그인" 탭
3. hong@example.com / password123 입력
4. "로그인하기" 클릭 ✅

5. 결과:
   📁 업무 폴더
   ├─ 📝 회의록
   └─ 📝 TODO 리스트

   ✅ 이전에 작성한 메모가 그대로!
   ⚠️ 김철수의 메모는 보이지 않음!
```

```javascript

  "local-users": [
    { uid: "local-1234", email: "hong@example.com", ... },
    { uid: "local-5678", email: "kim@example.com", ... }
  ],
  "local-current-session": { uid: "local-1234", ... },
  
  // 각 사용자별 메모 (uid로 분리)
  "memo-app-notes-local-1234": [ /* 홍길동의 메모 */ ],
  "memo-app-notes-local-5678": [ /* 김철수의 메모 */ ],

  "memo-app-folders-local-5678": [ /* 김철수의 폴더 */ ]
}

## 🎓 사용 팁

### 💡 여러 계정 사용하기
1. 업무용 계정과 개인용 계정을 분리해서 사용하세요
2. 각 계정에서 로그아웃 후 다른 계정으로 전환 가능
3. 모든 데이터는 안전하게 분리 저장됩니다

### 💡 데이터 백업 (로컬 모드)
- 브라우저의 로컬 스토리지에 저장됩니다
- 브라우저 데이터를 삭제하면 메모도 삭제됩니다
- Firebase 모드로 전환하면 클라우드 백업 가능

### 💡 비밀번호 관리
- 로컬 모드: 평문으로 저장 (데모용)
- Firebase 모드: Firebase가 안전하게 암호화 관리

## 🚀 시작하기

```

## ❓ 자주 묻는 질문

### Q: 여러 계정을 만들 수 있나요?
A: 네! 원하는 만큼 계정을 만들 수 있습니다.

### Q: 각 계정의 메모는 분리되나요?
A: 완전히 분리됩니다. 다른 사용자의 메모는 절대 보이지 않습니다.

### Q: 로그아웃하면 메모가 사라지나요?
A: 아니요. 다시 로그인하면 그대로 있습니다.

### Q: Firebase 설정이 필요한가요?
A: 아니요. 로컬 모드로 바로 사용 가능합니다. Firebase는 클라우드 동기화를 원할 때만 설정하세요.

### Q: 데이터는 안전한가요?
A: 
- 로컬 모드: 브라우저에만 저장 (다른 사람은 접근 불가)
- Firebase 모드: Google의 보안 시스템으로 보호
